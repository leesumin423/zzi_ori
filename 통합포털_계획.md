# 통합 자금포털 (허브형 포털) — 현황 및 TODO

> 이 문서는 "허브형 통합포털"(주가·공시 / 자금(차입) / 퇴직연금을 한 화면에서 보는 프로젝트)의
> 현재 상태와 남은 작업을 정리한 것입니다. `주가현황`(이 저장소, `zzi_ori`) 앱의 "공시 포털" 계획은
> [포털_배포_계획.md](포털_배포_계획.md)에 별도로 있으니 혼동하지 마세요 — **이 문서는 그것과는
> 다른, 4개 앱 전체를 하나로 묶는 상위 프로젝트**입니다.

## ⚠️ 먼저 알아야 할 것: 이 저장소(`zzi_ori`)엔 `주가현황`만 들어있습니다

통합포털은 아래처럼 **서로 다른 로컬 폴더에 있는 4개의 독립 프로그램**을 허브가 iframe으로
묶어서 보여주는 구조입니다. 이 GitHub 저장소(`zzi_ori`)는 그중 `주가현황` 폴더 하나만
추적(git)하고 있고, 나머지 3개(`차입금관리`, `퇴직연금_보고시스템`, `통합포털`)는 **git으로
관리되지 않는 로컬 폴더**입니다. 다른 개발툴로 이어서 작업하려면:

- **같은 PC에서 로컬 폴더를 그대로 열어서** 작업하거나 (`C:\Users\tyinc\Desktop\` 아래),
- 이 저장소를 clone 하는 것만으로는 `주가현황` 외 3개 앱의 코드를 받을 수 없다는 점을
  꼭 알고 있어야 합니다 (필요하면 나중에 이 3개 폴더도 별도 git 저장소로 만들 수 있습니다).

```
C:\Users\tyinc\Desktop\
├── ai 개발관련\
│   ├── 주가현황\              ← 이 저장소(zzi_ori git 연결됨). 포트 5000 (Flask, server.py)
│   ├── 차입금관리\             ← git 없음. 포트 8501 (Streamlit, app.py)
│   └── 통합포털\               ← git 없음. 포트 9000 (Flask 허브, app.py) — 로그인/승인/탭 관리
└── 퇴직연금_보고시스템\        ← git 없음. 포트 8000 (정적 HTML, run_dashboard.py)
```

## 아키텍처 (허브 + iframe)

- `통합포털/app.py` (Flask, 포트 9000)가 **로그인 · 계정승인 · 상단 탭 메뉴**만 담당하고,
  나머지 4개 화면은 각자 원래 포트로 계속 돌아가는 걸 **iframe**으로 그대로 보여줍니다.
- 각 서브앱은 새로 만들지 않고 **기존 코드를 그대로 재사용** — 허브는 그 위에 얹는 로그인 게이트일
  뿐입니다.
- 탭 구성: `📈 주가` / `📋 공시` / `💰 자금`(하위: 차입·대여·운용·관리) / `🧓 퇴직연금`
  - "차입"만 실제 화면 연결(기존 차입금관리 앱), 대여/운용/관리는 아직 화면 자체가 없어
    "준비 중" 안내만 표시(placeholder) — 화면이 생기면 `통합포털/app.py`의 `portal()` 라우트에서
    해당 탭의 `type`을 `'iframe'`으로 바꾸고 `url`만 채우면 됩니다.

### 2. 통합포털 허브 (로그인/승인/관리자)
- `통합포털/app.py`, `hub_db.py`(SQLite, `통합포털/data/hub.db`), `hub_config.py`
- 로그인 → 미승인자는 "접근 요청" → 관리자가 승인/거절 → 승인 시 계정 자동 생성.
- 관리자 화면(`/admin`)에서 계정 직접 생성, 승인/거절, 관리자 권한 부여, 계정 삭제 가능.
- 관리자 비밀번호(부트스트랩 계정): `hub_config.py`의 `BOOTSTRAP_ADMIN_USERNAME`/`PASSWORD`.

### 3. 차입금관리 데이터 저장 — 업로드 + 직접입력 이중화
- `차입금관리/snapshot_db.py`: 분기별 차입금 엑셀을 SQLite(`차입금관리/data/snapshots.db`)에
  통째로 저장해두는 기존 구조.
- `차입금관리/app.py`의 `⚙️ 데이터 관리` 화면에 **입력 방식 라디오(업로드/직접입력)** 추가:
  - 업로드: 기존 방식 그대로.
  - 직접입력: `build_loan_status_workbook()`이 `st.data_editor`로 편집한 표를 업로드본과
    똑같은 형식의 엑셀 바이트로 변환해 같은 DB에 저장 — 이후 로직은 입력 경로를 구분하지 않음.

### 4. 계정 시스템 — 그룹웨어 연동(테스트모드) + 팀 + 셀프서비스
- **팀명 필수**: 접근요청/계정생성 시 팀 입력 필수 (`users.team`, `access_requests.team` 컬럼).
  나중에 팀별 뷰어 권한을 나눌 때 이 컬럼을 그대로 쓰면 됩니다 (아래 "남은 작업" 참고).
- **그룹웨어ID 기반 가입 + 메일 발송**: 아이디 = 그룹웨어 아이디, 메일 = `아이디@tongyanginc.co.kr`
  (`hub_config.EMAIL_DOMAIN`). 승인/계정생성 시 초기 비밀번호를 이 메일로 발송.
  - **지금은 테스트 모드**: `hub_config.py`의 `SMTP_HOST`가 비어있으면 실제 발송 대신 관리자
    화면에 발송 내용을 그대로 보여줌. 실제 회사 메일 서버 정보(주소/포트/계정)를 알게 되면
    `hub_config.py`의 `SMTP_HOST`/`SMTP_PORT`/`SMTP_USER`/`SMTP_PASSWORD`만 채우면 코드 수정
    없이 실제 발송으로 전환됨 (`통합포털/mailer.py`).
- **비밀번호 찾기**: 로그인 화면 → 아이디 입력 → 6자리 인증코드를 메일로 발송(테스트모드는
  화면에 표시) → 코드 확인 후 재설정. 10분 유효 (`hub_db.py`의 `password_reset_codes` 테이블).
- **내 정보 페이지**: 로그인 후 `/account`에서 본인 정보 확인 + 비밀번호 변경(현재 비밀번호
  확인 필요).

## 남은 작업 (다음 개발자를 위한 파일 안내)

### A. 월간보고서 엑셀 export
첨부 참고파일(`2026년 06월 월간보고서_보고용_최종.xlsx`)의 **"차입금 및 담보현황" 시트** 서식
(구분/은행·지점/기간/금액(한도)/잔액/금리(%)/담보/비고)에 맞춰, 현재 로드된 차입금 데이터를
엑셀로 내보내는 기능. (다른 12개 시트는 데이터 소스가 아직 없어 범위에서 제외하기로 결정됨.)

- **작업 파일**: `차입금관리/app.py`
  - 참고할 기존 함수: `load_current_loan_status()`(약 553행) — 현재 차입금표를 DataFrame으로
    반환하는 함수, 이 결과를 그대로 재료로 쓰면 됨.
  - 참고할 기존 패턴: `build_loan_status_workbook()`(약 660행 부근, 이번에 추가됨) — openpyxl로
    워크북을 만드는 방식을 그대로 참고 가능(다만 이건 우리 내부 포맷이고, 이번 신규 기능은
    참고파일의 "구분/은행·지점/기간/금액(한도)/잔액/금리(%)/담보/비고" 8개 컬럼 헤더와 스타일을
    똑같이 맞춰야 함 — 병합 셀, 굵은 헤더 등 서식까지 재현하려면 openpyxl의 `cell.font`,
    `cell.border`, `merge_cells()` 활용 필요).
  - `🖨 인쇄용 요약` 페이지(`PAGE_` 상수로 검색하면 위치 찾기 쉬움) 근처에 "📥 엑셀로 내보내기"
    버튼을 추가하고 `st.download_button()`으로 제공하는 방식을 추천.

### B. 팀별 뷰어 권한 분리
지금은 `users.team`/`access_requests.team` 컬럼만 있고, 실제로 팀에 따라 보이는 화면(탭)을
제한하는 로직은 없습니다(전 계정이 전체 탭을 다 봄).

- **작업 파일**: `통합포털/app.py`
  - `portal()` 라우트(탭 목록 만드는 곳)에서 `user['team']`을 확인해 `tabs` 리스트를 필터링하면
    됨. 예: 특정 탭에 `'allowed_teams': ['자금팀']` 같은 필드를 추가하고, 목록을 만들 때
    `if not tab.get('allowed_teams') or user['team'] in tab['allowed_teams']`로 걸러내기.
  - 어떤 탭을 어떤 팀에게 허용할지는 **정책 확정이 필요** (아직 미정 — 사용자 확인 후 진행).
  - `통합포털/hub_db.py`는 이미 `team` 컬럼과 `list_users()`/`get_user_by_id()` 등에서 팀 값을
    반환하고 있으니 추가 스키마 변경은 필요 없을 가능성이 높음.
