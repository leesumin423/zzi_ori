# 데이터 출처 정리

이 대시보드가 화면에 뿌리는 모든 숫자가 **어디서, 어떻게** 오는지 정리한 문서입니다. 전부 [server.py](server.py) 안에 있고, 표의 "함수" 열에 적힌 이름으로 코드에서 바로 찾을 수 있습니다.

모든 데이터는 네이버 금융(finance.naver.com)의 **비공식 공개 API/페이지**를 그대로 가져다 쓰는 것이라, 네이버가 페이지 구조를 바꾸면 언제든 깨질 수 있습니다. 그럴 땐 서버 콘솔에 찍히는 `[DEBUG]` 로그를 보면 어느 항목이 실패했는지 알 수 있습니다.

## 한눈에 보는 요약표

| 화면 항목 | 데이터 출처(URL) | 방식 | 담당 함수 |
|---|---|---|---|
| USD/KRW 환율 | `api.stock.naver.com/marketindex/exchange/FX_USDKRW/prices` | JSON API | `fetch_exchange` |
| KOSPI/KOSDAQ 지수 | `polling.finance.naver.com/api/realtime/domestic/index/{code}` | JSON API | `fetch_index` |

| 테마별 등락률 | `finance.naver.com/sise/sise_group_detail.naver?type=theme&no=N` | HTML 스크래핑 | `fetch_theme_change` |

## 1. 환율 (USD/KRW)

- **출처**: `https://api.stock.naver.com/marketindex/exchange/FX_USDKRW/prices?page=1&pageSize=2`
- **형식**: 일자별 시세 JSON 배열 (배열 0번째가 가장 최근, 1번째가 그 전날)
- **사용 필드**: `closePrice`(종가), `fluctuations`(전일대비), `fluctuationsRatio`(등락률)
- **현재기준 / 장마감기준**: 외환은 24시간 거래되므로 "장마감"이 정확히 없음 → 배열 0번째(오늘 값, 계속 갱신)를 현재기준, 1번째(직전 영업일 확정치)를 장마감기준으로 사용

## 2. 지수 현황 (KOSPI / KOSDAQ)

- **출처**: `https://polling.finance.naver.com/api/realtime/domestic/index/{KOSPI|KOSDAQ}`
- **사용 필드**: `closePrice`(현재가 — 이름과 달리 장중엔 실시간으로 계속 바뀜), `compareToPreviousClosePrice`, `fluctuationsRatio`, `marketStatus`(OPEN 여부)
- **현재기준**: 위 API의 `closePrice`를 그대로 사용
- **장마감기준**: 별도로 `fchart.stock.naver.com/sise.nhn?symbol={code}&timeframe=day&count=10`(일봉 XML)을 받아서, 장이 열려있으면 "직전 확정 거래일"의 종가를, 닫혀있으면 당일 종가를 사용 — 그 하루 전 종가와 비교해 등락폭/색상을 별도 계산 (`fetch_index_close_basis`)
- **지수 카드 하단 문구** ("개인, 외인 순매도 / 기관 순매수" 등): 아래 3번 항목에서 만든 문자열을 붙인 것

## 3. 지수 투자자 동향 요약

- **출처**: `https://finance.naver.com/` 메인 페이지를 통째로 받아 HTML 파싱
- **방식**: `kospi_area`/`kosdaq_area` 클래스의 `<div>` 안에서 "개인/외국인/기관"이 모두 들어있는 `<dl>` 태그를 찾아 각 금액(억원)을 추출
- **가공**: 값이 음수면 "순매도", 양수면 "순매수"로 분류해 `"외인 순매도 / 기관 순매수"` 같은 문장으로 조립 (`investor_summary`)

## 4. 테마별 시세

- **출처**: `https://finance.naver.com/sise/sise_group_detail.naver?type=theme&no={번호}`
- **방식**: `td.number span` 셀렉터로 첫 번째 등락률 값을 추출
- **현재 사용 중인 테마 번호**

  | 테마 | no |
  |---|---|
  | 시멘트/레미콘 | 44 |
  | 건설대표주 | 154 |
  | 증권 | 151 |
  | 미디어(방송/신문) | 232 |

  다른 테마로 바꾸려면 해당 테마의 네이버 페이지 URL에서 `no=` 뒤 숫자를 확인해 `server.py`의 `THEMES` 리스트만 고치면 됩니다.

## 5. 개별 종목 — 계열사 / 시멘트사 표

한 종목당 아래 3개 출처를 조합해서 만듭니다.

| 값 | 출처 | 비고 |
|---|---|---|
| 당일 주가(현재기준) | `itemSummary.nhn`의 `now` | 장중 실시간, 장마감 후엔 자동으로 종가 고정 |
| 당일 주가(장마감기준) | `now - diff`(장중) 또는 `now`(장마감 후) | `diff`도 같은 API의 필드. 장 시간 판정은 `is_krx_open()`(평일 09:00~15:30, 공휴일 미반영) |
| 전년말 주가 | `fchart` XML 540봉 중 전년 12월 마지막 거래일 종가 | 연도가 바뀌면 `fetch_stock` 안의 `last_year = date.today().year - 1` 로직이 자동으로 따라감 |
| 52주 최고/최저 | `item/main.naver` 페이지의 "52주최고l최저" 표시값 그대로 | 예전엔 fchart 540봉(2년치) 안에서 직접 max/min을 계산했는데, 그러면 진짜 52주(1년)보다 넓은 범위 값이 섞여 네이버 표시값과 달라지는 버그가 있어서 지금은 네이버가 이미 계산해놓은 값을 그대로 긁어옴 |
| 상장주식수 | `item/main.naver` 페이지에서 "상장주식수" 라벨 옆 값 | 정규식 2단계 폴백 포함 |
| 시가총액 | 주식수 × 주가 ÷ 1억 | 서버에서 직접 계산 (현재기준/장마감기준 각각) |
| 등락율(표의 "등락율" 컬럼) | (당일 시가총액 − 전년말 시가총액) ÷ 전년말 시가총액 × 100 | 전일대비가 아니라 **전년말 대비**임에 유의 |
| 자본금(억원) | `server.py`의 `CAPITAL`/`CEMENT_CAPITAL` 딕셔너리 | API로 못 가져와서 **수동 입력** (DART 공시 기준) |

## 6. 수급 동향 모달 (회사명 클릭 시 팝업)

- **출처**: `https://finance.naver.com/item/frgn.naver?code={종목코드}`
- **표 선택 방식**: 페이지 안에 표가 여러 개 있고 종목마다 개수/순서가 달라서(투자유의종목 등은 표가 하나 더 붙어 밀림), 고정 인덱스 대신 표의 `summary` 속성에 "외국인"과 "순매매"가 모두 들어있는 표를 찾아서 사용
- **원본 제공 값**: 날짜, 종가, 등락률, 거래량, **기관 순매매(주식 수)**, **외국인 순매매(주식 수)**
- **계산해서 채우는 값** (네이버가 이 페이지에서 개인·거래대금은 안 주기 때문)
  - **총거래대금(백만원)** = 거래량 × 종가
  - **개인순매매(추정)** = −(기관순매매 + 외국인순매매) — 기타법인 등은 무시한 근사치. **실측값이 아니라 잔차 추정치**이므로 참고용으로만 볼 것

## 7. 주가 동향 코멘트 (표 아래 자동 생성 문단)

동양/유진기업 대상으로, 최근 10영업일 데이터를 분석해 문장을 자동 생성합니다 (`generate_stock_commentary`). 순서대로:

1. **연속 순매수/순매도 스트릭** — 위 6번 수급 데이터에서 기관·외국인 순매매 부호가 3일 이상 연속되면 언급
2. **당일 주가 변동 서술** — `fetch_daily_ohlc`로 받은 일별 시가/고가/저가/종가(fchart) 중, **전일 종가 대비 장중 고점·저점 도달폭**이 가장 큰 날을 찾음(`_find_notable_intraday_day`)
   - 그 도달폭이 종가 기준 등락률보다 훨씬 크면(장중에 급등락 후 되돌림이 있었던 경우) "시가 X원 → 장중 한때 Y원 → 종가 Z원" 형태로 상세 서술 + 그날의 개인/기관/외국인 수급 요약(`_flow_summary_sentence`) + 관련 뉴스 헤드라인
   - 그렇지 않으면(예: 상한가로 마감 등) 단순 "종가 기준 급등락 + 뉴스"로 대체
3. **초저유동성 상태** — 최근 거래대금이 계속 0(무거래)이면 마지막에 언급

- **뉴스 헤드라인 출처**: `https://finance.naver.com/item/news_news.naver?code={종목코드}` (같은 날짜 + 종목명이 제목에 포함된 기사를 우선 매칭, `_match_news`)
- ⚠️ 이 코멘트는 공개 데이터로 자동 추정한 것이라 **투자 판단 근거로 쓰면 안 되고**, 원인 설명이 실제 확인된 뉴스가 없으면 "관련 뉴스는 확인되지 않음"이라고 정직하게 표시됩니다.
