# 데이터 출처 정리

이 대시보드가 화면에 뿌리는 모든 숫자가 **어디서, 어떻게** 오는지 정리한 문서입니다. 전부 [server.py](server.py) 안에 있고, 표의 "함수" 열에 적힌 이름으로 코드에서 바로 찾을 수 있습니다.

모든 데이터는 네이버 금융(finance.naver.com)의 **비공식 공개 API/페이지**를 그대로 가져다 쓰는 것이라, 네이버가 페이지 구조를 바꾸면 언제든 깨질 수 있습니다. 그럴 땐 서버 콘솔에 찍히는 `[DEBUG]` 로그를 보면 어느 항목이 실패했는지 알 수 있습니다.

## 1. 환율 (USD/KRW)

- **출처**: `https://api.stock.naver.com/marketindex/exchange/FX_USDKRW/prices?page=1&pageSize=2`
- **형식**: 일자별 시세 JSON 배열 (배열 0번째가 가장 최근, 1번째가 그 전날)
- **사용 필드**: `closePrice`(종가), `fluctuations`(전일대비), `fluctuationsRatio`(등락률)
- **현재기준 / 장마감기준**: 외환은 24시간 거래되므로 "장마감"이 정확히 없음 → 배열 0번째(오늘 값, 계속 갱신)를 현재기준, 1번째(직전 영업일 확정치)를 장마감기준으로 사용

## 2. 지수 현황 (KOSPI / KOSDAQ)

- **출처**: `https://polling.finance.naver.com/api/realtime/domestic/index/{KOSPI|KOSDAQ}`
- **사용 필드**: `closePrice`(현재가 — 이름과 달리 장중엔 실시간으로 계속 바뀜), `compareToPreviousClosePrice`, `fluctuationsRatio`, `marketStatus`(OPEN 여부)
- **현재기준**: 위 API의 `closePrice`를 그대로 사용
- **장마감기준**: 별도로 `fchart.stock.naver.com/sise.nhn?symbol={code}&timeframe=day&count=10`(일봉 XML)을 받아서, 장이 열려있으면 "직전 확정 거래일"의 종가를, 닫혀있으면 당일 종가를 사용 — 그 하루 전 종가와 비교해 등락폭/색상을 별도 계산 (`fetch_index_close_basis`)
- **지수 카드 하단 문구** ("개인, 외인 순매도 / 기관 순매수" 등): 아래 3번 항목에서 만든 문자열을 붙인 것

## 3. 지수 투자자 동향 요약

- **출처**: `https://finance.naver.com/` 메인 페이지를 통째로 받아 HTML 파싱
- **방식**: `kospi_area`/`kosdaq_area` 클래스의 `<div>` 안에서 "개인/외국인/기관"이 모두 들어있는 `<dl>` 태그를 찾아 각 금액(억원)을 추출
- **가공**: 값이 음수면 "순매도", 양수면 "순매수"로 분류해 `"외인 순매도 / 기관 순매수"` 같은 문장으로 조립 (`investor_summary`)

## 4. 테마별 시세

- **출처**: `https://finance.naver.com/sise/sise_group_detail.naver?type=theme&no={번호}`
- **방식**: `td.number span` 셀렉터로 첫 번째 등락률 값을 추출
- **현재기준 / 장마감기준**: 이 네이버 페이지는 그 순간의 등락률 하나만 주고, 개별
  종목처럼 "오늘/전일"을 함께 주는 구조도 아니고 테마 전용 과거 이력 API도 없음 —
  그래서 서버가 직접 날짜 경계로 캐싱한다. **KRX 정규장 마감(15:30) 이후(주말
  포함)** 조회한 값을 "그날의 확정치"로 메모리에 저장해뒀다가, **다음 장중**엔
  그 캐시값을 장마감기준으로, 그 순간의 실시간 값을 현재기준으로 보여준다
  (`fetch_theme_change`/`_theme_daily_cache`). 서버가 재시작되면 캐시가 비어
  첫 조회 시엔 현재기준=장마감기준으로 동일하게 나오다가, 그날 마감을 한 번
  지나면 그 다음부터 정상적으로 갈라짐 — 다른 인메모리 캐시들과 동일한 한계.
- **현재 사용 중인 테마 번호**

  | 테마 | no |
  |---|---|
  | 시멘트/레미콘 | 44 |
  | 건설대표주 | 154 |
  | 증권 | 151 |
  | 미디어(방송/신문) | 232 |

  다른 테마로 바꾸려면 해당 테마의 네이버 페이지 URL에서 `no=` 뒤 숫자를 확인해 `server.py`의 `THEMES` 리스트만 고치면 됩니다.

## 5. 개별 종목 — 계열사 / 시멘트사 표

한 종목당 아래 3개 출처를 조합해서 만듭니다.

| 값 | 출처 | 비고 |
|---|---|---|
| 당일 주가(현재기준) | `itemSummary.nhn`의 `now` — 단, **NXT(넥스트레이드) 프리마켓(08:00~09:00)ㆍ애프터마켓(15:30~20:00) 시간대**엔 `item/main.naver` 페이지의 `id="rate_info_nxt"` 블록 값으로 대체 | `is_nxt_extended_hours()`가 이 시간대 여부를 판단. `item/main.naver` 페이지엔 KRX용(`rate_info_krx`)ㆍNXT용(`rate_info_nxt`) 시세 블록이 항상 둘 다 서버사이드에 렌더링돼 있고 탭 전환은 순수 화면 토글이라, 굳이 별도 폴링 API를 안 써도 이 페이지 하나로 두 값을 동시에 얻을 수 있음(`_parse_rate_info_block`). NXT 비대상 종목이면 자동으로 KRX 값 그대로 사용. 응답의 `current.source`가 "KRX"/"NXT" 중 어느 걸 썼는지 알려주고, 화면엔 NXT일 때만 작은 "NXT" 배지가 붙음 |
| 당일 주가(장마감기준) | `now - diff`(장중) 또는 `now`(장마감 후) | `diff`도 같은 API의 필드. 장 시간 판정은 `is_krx_open()`(평일 09:00~15:30, 공휴일 미반영). 이 값은 **KRX 정규장 15:30 종가 고정**이라 NXT 시간대여도 안 바뀜(의도된 동작 — "장마감기준"은 항상 KRX 정규장 마감가를 뜻함) |
| 전년말 주가 | `fchart` XML 540봉 중 전년 12월 마지막 거래일 종가 | 연도가 바뀌면 `fetch_stock` 안의 `last_year = date.today().year - 1` 로직이 자동으로 따라감 |
| 52주 최고/최저 | `item/main.naver` 페이지의 "52주최고l최저" 표시값 그대로 | 예전엔 fchart 540봉(2년치) 안에서 직접 max/min을 계산했는데, 그러면 진짜 52주(1년)보다 넓은 범위 값이 섞여 네이버 표시값과 달라지는 버그가 있어서 지금은 네이버가 이미 계산해놓은 값을 그대로 긁어옴 |
| 상장주식수 | `item/main.naver` 페이지에서 "상장주식수" 라벨 옆 값 | 정규식 2단계 폴백 포함 |
| 시가총액 | 주식수 × 주가 ÷ 1억 | 서버에서 직접 계산 (현재기준/장마감기준 각각) |
| 등락율(표의 "등락율" 컬럼) | (당일 시가총액 − 전년말 시가총액) ÷ 전년말 시가총액 × 100 | 전일대비가 아니라 **전년말 대비**임에 유의 |
| 자본금(억원) | DART 재무제표 API `fnlttSinglAcntAll.json`(개별, 최근 사업보고서)의 "자본금" 계정 | `fetch_dart_capital`. `DART_API_KEY` 없으면 `CAPITAL_FALLBACK` 딕셔너리(하드코딩, 부정확할 수 있음)로 대체 |

## 6. 수급 동향 모달 (회사명 클릭 시 팝업)

- **출처**: `https://finance.naver.com/item/frgn.naver?code={종목코드}`
- **표 선택 방식**: 페이지 안에 표가 여러 개 있고 종목마다 개수/순서가 달라서(투자유의종목 등은 표가 하나 더 붙어 밀림), 고정 인덱스 대신 표의 `summary` 속성에 "외국인"과 "순매매"가 모두 들어있는 표를 찾아서 사용
- **원본 제공 값**: 날짜, 종가, 등락률, 거래량, **기관 순매매(주식 수)**, **외국인 순매매(주식 수)**
- **계산해서 채우는 값** (네이버가 이 페이지에서 개인·거래대금은 안 주기 때문)
  - **총거래대금(백만원)** = 거래량 × 종가
  - **개인순매매(추정)** = −(기관순매매 + 외국인순매매) — 기타법인 등은 무시한 근사치. **실측값이 아니라 잔차 추정치**이므로 참고용으로만 볼 것

## 7. 주가 동향 코멘트 (표 아래 자동 생성 문단)

동양/유진기업 대상으로, 최근 10영업일 데이터를 분석해 문장을 자동 생성합니다 (`generate_stock_commentary`). 순서대로:

1. **당일(가장 최근 실제 거래일) 동향** — 변동폭 크기와 무관하게 항상 서술 (`_describe_latest_trading_day`)
   - `fetch_daily_ohlc`(fchart 일봉)로 시가 → (전일 종가 대비 장중 고점/저점이 종가보다 훨씬 튀면) 장중 급변동 → 종가 흐름을 서술
   - 그날의 개인(추정)/기관/외국인 수급 중 매수·매도 우위를 원인으로 붙임 (`_flow_summary_sentence`)
   - 관련 뉴스 헤드라인을 붙임 — 같은 날짜에 없으면 최근 3일 내로 확대 탐색 (`_find_related_news`)
2. **연속 순매수/순매도 스트릭** — 위 6번 수급 데이터에서 기관·외국인 순매매 부호가 3일 이상 연속되면 언급
3. **거래정지 여부** — 최근 며칠간 거래대금이 계속 0이면, `fetch_trade_status`(실시간 API)로 실제 거래정지인지 확인. 맞으면 8번 항목(DART)에서 사유를 찾아 붙이고, 아니면 단순 "초저유동성"으로 표시

- ⚠️ 이 코멘트는 공개 데이터로 자동 추정한 것이라 **투자 판단 근거로 쓰면 안 되고**, 확인되지 않은 원인은 "확인되지 않음"이라고 정직하게 표시됩니다.
- **뉴스 매칭 시 주의**: 회사명이 흔한 낱말이면(예: "동양") 그 날짜에 태그된 아무 기사나 잡히지 않도록, 회사명(또는 접미사를 뗀 핵심 이름 — `_company_root`, 예: 유진기업→유진)이 제목에 실제로 있는 기사만 인용합니다.

## 8. DART(전자공시) 연동

저장소에 포함된 팀 공용 `.dart_api_key`로 기본 동작합니다. 키가 없거나 조회에 실패하면 아래 기능들은 각자의 폴백 값/추정으로 자동 대체됩니다. 키 발급·교체 방법은 [README.md](README.md) 참고.

### 8-1. 거래정지 등 사유 확인 (7번 코멘트에서 사용)

- **공시 목록**: `https://opendart.fss.or.kr/api/list.json` — `corp_code`(종목코드가 아닌 DART 전용 고유번호, `DART_CORP_CODES` 딕셔너리로 매핑), `bgn_de`/`end_de`(조회 시작·종료일, 최근 60일) 파라미터로 조회 (`fetch_dart_disclosures`)
- **공시 원문**: `https://opendart.fss.or.kr/api/document.xml` — 공시 목록에서 "주식병합결정"/"주식소각결정" 제목을 찾아 해당 건의 `rcept_no`로 원문(zip 안의 XML/HTML)을 받아옴 (`fetch_dart_document_text`)
- **원문에서 뽑는 값**: "매매거래정지기간"의 시작일/종료일을 표 구조(rowspan)를 따라가며 파싱 (`_extract_date_range_section`) — 정확한 정지 시작/종료일을 그대로 인용
- 예시(2026.07 동양): "단기과열종목 지정 연장" 공시가 시점상 가장 최근이라 예전엔 이게 정지 사유로 잘못 인용됐었는데, DART 원문을 보니 실제 원인은 "[기재정정]주식병합결정"(2026.06.02, 매매거래정지기간 2026-07-03~2026-07-16)이었음 — 네이버 공시 게시판은 제목만 있고 본문이 KIND/DART로 JS 리다이렉트되어 있어 이건 DART API 없이는 확인이 안 됐음

### 8-2. 자본금 (계열사/시멘트사 표에서 사용)

- **출처**: `https://opendart.fss.or.kr/api/fnlttSinglAcntAll.json` — 개별재무제표(`fs_div=OFS`), 최근 사업보고서(`reprt_code=11011`)에서 `sj_div=BS`(재무상태표) & `account_nm=자본금`인 행의 `thstrm_amount`를 억원 단위로 변환 (`fetch_dart_capital`)
- 연초라 직전 연도 사업보고서가 아직 안 나온 경우를 대비해 최대 2개 연도(작년→재작년) 재시도
- 서버가 켜져 있는 동안은 종목별로 한 번 조회한 값을 메모리에 캐싱 (`_dart_capital_cache`) — 자본금은 거의 안 바뀌는 값이라 매 새로고침마다 다시 조회할 필요가 없음
- **예전엔 종목마다 "보통주 × 액면가"를 손으로 추정 계산해서 하드코딩**했었는데, 액면가가 종목마다 다르다는 걸 놓쳐서 실제 공시와 크게 어긋난 값이 있었음 (한일시멘트 310억→실제 368억, 성신양회 300억→실제 1,285억). DART 실측값으로 바꾸면서 해결됨
