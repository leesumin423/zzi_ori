## 1. 공시 모니터링

> DART Open API를 연동해 기존에 수기 엑셀로 관리하던 공시 현황을 자동화한 기능입니다.  
> 주가현황 대시보드 사이드바의 **"공시"** 메뉴 안에 서브탭으로 붙어 있습니다.

---

### 단판공시 (단일판매·공급계약체결)

**목적**: 동양(주)이 신고하는 수주 공시를 자동 집계해, 현재 진행 중인 현장만 표로 정리  
**문서**: [`단판공시_모니터링.md`](단판공시_모니터링.md)

#### 어떤 문제를 해결했나

| 기존 방법 | 자동화 후 |
|---|---|
| DART에서 공시 건별 수동 검색 | DART API로 전건 자동 수집 |
| 엑셀 수기 취합 (협조전발송용 파일) | 실시간 조회 → 웹 화면으로 표시 |
| 준공 현장 정리 누락 가능 | 정기보고서 "진행현황" 기준으로 자동 필터 |

#### 어떤 자료를 쓰나

- **DART Open API** — `단일판매ㆍ공급계약체결` 공시 원문 전체
- **정기보고서(사업/반기/분기)** — "단판공시에 대한 진행현황" 표 (현장 종료 판단 1차 기준)
- 팀 관리 참고 엑셀 (광양바이오매스·보령발전본부·국회대로 금액 대조 검증 완료)

#### 무엇을 보여주나

| 컬럼 | 설명 |
|---|---|
| 현장명 | 공시상 사업명 |
| 계약상대방 | 발주처 |
| 최초계약일 | 최초 신고 기준 |
| 최근공시일 (정정횟수) | 가장 최근 신고일 + 정정 횟수 |
| 계약금액 (백만원) | 최근 공시 기준 |
| 변동율 (최초 대비) | 최초 계약금액 대비 변동 % |
| 공사기간 | 시작일 ~ 종료일 |
| 원문 | DART 원문 링크 |

- **사전검증 계산기**: 계약(예정)일자 + 계약금액을 입력하면 공시 의무 대상 여부 즉시 판단
- **규정 보기 모달**: 해당 연도 연결 매출액 × 5% 기준금액 실시간 계산

#### 개발 상태

- ✅ 1단계: 현장 자동 스크랩 + 진행 현장 필터링 + 사전검증 계산기
- ⬜ 2단계 (예정): 엑셀 다운로드, 매월 자동 메일 발송, 다른 계열사 확장

### 공정위 신고 · 지분공시

**지분공시 목적**: 임원·주요주주의 자사주 매수 이력을 DART에서 자동 집계 (기존 수기 엑셀 대체)  
**문서**: [`지분공시_모니터링.md`](지분공시_모니터링.md)

#### 어떤 문제를 해결했나

| 기존 방법 | 자동화 후 |
|---|---|
| DART 공시 건별 수동 검색 후 엑셀 취합 | DART API로 10년치 자동 수집 |
| 임원별 누적 주식수 수기 계산 | 최근 공시 "변동후" 확정값 자동 표시 |
| 퇴임 임원 정리 누락 가능 | 현직 임원만 자동 필터 (주요주주 법인 예외) |

#### 어떤 자료를 쓰나

- **DART Open API** — `임원ㆍ주요주주특정증권등소유상황보고서` (최근 10년, 장내매수 건만)
- **대량보유상황보고서** (5% Rule) — 별도 하위탭으로 구분
- 팀 관리 참고 엑셀 (황이석·정진학·안준수·황부익 수치 대조 검증 완료)

#### 무엇을 보여주나

**① 임원·주요주주 지분공시 이력 탭**

| 컬럼 | 설명 |
|---|---|
| 이름 | 임원/주주명 (가나다순) |
| 구분 | 사내이사 / 사외이사 / 미등기 / 최대주주 |
| 최초 매수일 | 10년 내 첫 장내매수 일자 |
| 최근 매수일 (공시일) | 가장 최근 거래 신고일 |
| 누적 주식수 | 최근 공시 "변동후" 최종 보유량 |
| 평균단가 (원) | 거래별 가중평균 |
| 원문 | DART 원문 링크 |

**② 대량보유상황보고서 (5% Rule) 탭** — 5% 이상 대량보유자의 신고 이력

#### 개발 상태

- ✅ 1단계: 이력 자동 집계 + 현직 임원 필터 + 규정 보기 모달
- ⬜ 2단계 (예정): 공정위 기업결합 신고 연동 (착수 전)

## 2. 주가현황 대시보드

**목적**: 네이버 금융 데이터를 긁어와 계열사·시멘트사 주가를 한 화면 다크모드로 정리, PDF 보고서 출력  
**파일**: `server.py` · `주가현황.html` · `dashboard.js` · `styles.css`

> 개발 지식이 없는 분은 [`dist/주가현황_배포.zip`](dist/주가현황_배포.zip)만 받아서 압축 해제 후 `주가현황_실행하기.bat` 더블클릭

#### 어떤 자료를 쓰나

| 데이터 | 출처 | 비고 |
|---|---|---|
| KOSPI / KOSDAQ 지수 | 네이버 금융 (비공식 API) | |
| 계열사·시멘트사 주가 | 네이버 금융 | 실시간 / 장마감 기준 전환 가능 |
| 테마별 시세 | 네이버 금융 테마 그룹 | 시멘트/레미콘, 건설대표주, 증권 등 |
| 수급 동향 (5일) | 네이버 금융 | 기관·외국인·개인(추정) 순매매 |
| 자본금 | DART 재무제표 API | 실측값 자동 조회, 실패시 하드코딩 폴백 |
| 거래정지 사유 | DART 공시 원문 | 주식병합·소각 결정 공시 파싱 |
| 뉴스 헤드라인 | 네이버 금융 뉴스 | 주가 동향 코멘트 매칭용 |
| USD/KRW 환율 | 네이버 금융 | |

상세 출처 및 API 엔드포인트: [`데이터_출처.md`](데이터_출처.md)

#### 주요 기능

| 기능 | 설명 |
|---|---|
| 지수 현황 | KOSPI/KOSDAQ 현재가 + 투자자 동향 요약 |
| 테마별 시세 | 시멘트/레미콘, 건설대표주, 증권, 미디어 등 등락률 |
| 계열사·시멘트사 주가 | 주식수·자본금·전년말 대비 시가총액·52주 최고/최저 |
| 현재기준/장마감기준 탭 | 실시간가 vs 15:30 확정 종가 전환 |
| 수급 동향 모달 | 회사명 클릭 → 최근 5일 수급 팝업 |
| 주가 동향 코멘트 | 10영업일 수급 패턴 분석 + 뉴스 매칭 → 보고서 톤 자동 생성 |
| Export PDF | 브라우저 인쇄 기능으로 그대로 PDF 저장 |
| 공시 메뉴 | 단판공시·지분공시 (위 1번 항목, 별도 서브탭) |

#### 커스터마이징 위치 (`server.py`)

| 바꾸고 싶은 것 | 위치 |
|---|---|
| 계열사 종목·종목코드 | `COMPANIES` 딕셔너리 |
| 계열사 자본금 | DART 자동 조회 (`DART_CORP_CODES`에 고유번호 추가 필요) |
| 시멘트사 종목 | `CEMENT` 딕셔너리 |
| 테마 그룹 | `THEMES` 리스트 (네이버 테마 `no=` 번호) |
| 주가 코멘트 대상 | `TARGETS` 리스트 |
| 거래정지 대체 문구 | `MANUAL_HALT_REASONS` 딕셔너리 |

#### 실행 방법

```bash
# 1. 패키지 설치
pip install -r requirements.txt

# 2. 서버 실행
python server.py

# 3. 브라우저에서 접속
http://localhost:5000
```

Windows: `주가현황_실행하기.bat` 더블클릭 (서버 실행 + 브라우저 열기 자동)

#### DART API 키

`.dart_api_key` 파일에 팀 공용 키가 포함되어 있어 **별도 설정 없이 바로 동작**합니다.  
한도 초과 시 [opendart.fss.or.kr](https://opendart.fss.or.kr)에서 개인 키 무료 발급 가능.

> ⚠️ 이 저장소는 **Private**으로 유지하세요. 키가 평문으로 포함되어 있어 Public으로 전환하면 외부에 노출됩니다.

#### 알려진 제한사항

- 네이버 금융 비공식 API 사용 → 네이버 구조 변경 시 일부 항목 장애 가능 (콘솔 `[DEBUG]` 로그 확인)
- 52주 최고/최저, 전년말 주가는 거래일 기준으로 소폭 차이 날 수 있음
- 환율은 USD/KRW만 지원
- 개인 순매매는 추정치 (기관·외국인 잔차로 역산)

## 3. 그룹웨어 업무 체크리스트

**목적**: 그룹웨어 메일함·전자결재함을 자동 조회해 마감일 언급 항목을 추출, 매일 아침 메모지 팝업으로 표시  
**레포**: [leesumin423/-AI-](https://github.com/leesumin423/-AI-) → `groupware-task-checklist/`

#### 어떤 문제를 해결했나

| 기존 방법 | 자동화 후 |
|---|---|
| 매일 아침 그룹웨어 수동 로그인 | Windows 작업 스케줄러 07:40 자동 실행 |
| 메일함 스크롤하며 마감 수동 확인 | 마감 키워드 문장 → 날짜 자동 추출 |
| 전자결재 4개 함 별도 확인 | 메일함 + 결재 4함 동시 조회 |
| 날짜 계산 머릿속으로 | 지남/오늘/이번주/차주 자동 분류 |
| 완료 여부 기억 | 체크 후 저장 → 다음 날 자동 제외 |

#### 어떤 자료를 쓰나

| 조회 대상 | 설명 |
|---|---|
| 받은메일함 | 마감 키워드가 있는 메일 본문 |
| 전자결재 부서함 완료함 | 부서 결재 완료 문서 |
| 전자결재 부서함 참조/회람함 | 부서 참조 문서 |
| 전자결재 개인함 완료함 | 개인 결재 완료 문서 |
| 전자결재 개인함 참조/회람함 | 개인 참조 문서 |

#### 어떻게 동작하나

```
07:40 자동 실행
  → 로그인 세션 확인 (만료 시 브라우저 열어 수동 로그인 유도 → 완료 후 자동 계속)
  → 메일함·전자결재함 헤드리스 크롤링 (Playwright)
  → "까지/마감/제출/회신" 등 키워드 문장에서 날짜 추출
  → 지남 / 오늘 / 이번주 / 차주 / 월별 분류
  → 메모지 팝업 (tkinter) 표시
  → 완료 체크 저장 → 다음 실행 시 제외
```

#### 기술 스택

| 역할 | 도구 |
|---|---|
| 웹 크롤링 | Playwright (Python) |
| 날짜·텍스트 파싱 | `re`, `datetime` (표준 라이브러리) |
| UI 팝업 | `tkinter` (표준 라이브러리) |
| 설정 | `python-dotenv` |
| 자동 실행 | Windows 작업 스케줄러 |

### 🌐 포털 배포 시 추가 확인 (해당 시)

> 현재 포털 배포는 **보류 중** ([포털_배포_계획.md](포털_배포_계획.md) 참고). 착수 시 아래 항목 추가 확인.

- [ ] Flask 개발 서버(`app.run(debug=True)`) → **waitress** 프로덕션 서버로 교체
- [ ] 서버 PC 재부팅 시 자동 시작 (Windows 서비스 또는 작업 스케줄러 등록)
- [ ] 사내망 IP 대역 외 접근 차단 (방화벽 설정)
- [ ] 단판공시(사업부서 공유 가능) / 지분공시(자금팀 전용) 탭 접근 권한 분리 구현
- [ ] HTTPS 적용 여부 결정 (그룹웨어가 HTTPS면 iframe 삽입 시 http 콘텐츠 차단 이슈)
- [ ] 그룹웨어 메뉴 링크 추가 또는 iframe 삽입 가능 여부 그룹웨어 담당자 확인

### ✅ 기능 검증

- [ ] 단판공시 — 팀 관리 엑셀과 현장 목록 및 계약금액 대조 확인
- [ ] 지분공시 — 팀 관리 엑셀과 임원별 누적 주식수·평균단가 대조 확인
- [ ] 주가 동향 코멘트 — 거래정지 종목이 있을 경우 사유 문구 정상 표시 확인
- [ ] Export PDF — A4 인쇄 시 표가 잘리거나 페이지 밖으로 넘치지 않는지 확인
- [ ] 그룹웨어 체크리스트 — 마감 항목이 올바른 버킷(오늘/이번주/차주)으로 분류되는지 확인
- [ ] 그룹웨어 체크리스트 — 완료 체크 후 다음 실행 시 해당 항목이 제외되는지 확인
